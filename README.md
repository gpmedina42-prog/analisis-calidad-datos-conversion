# ¿Cuánto dinero pierde una operación por mala calidad de datos?

Análisis estratégico real con datos protegidos mediante **masking en SQL**.

## 4 Palancas de Mejora Identificadas

**Calidad de base:** 55.6% de registros sin datos con efectividad 38% menor (1.49% vs 2.05%).  
Impacto estimado: +309 ventas si toda la base fuera completa.

**Speech diferenciado:** La contactabilidad es uniforme — el quiebre ocurre después del contacto.  
Impacto estimado: +105 ventas adicionales.

**Contactabilidad:** Mejora de 3 puntos porcentuales representa +87 ventas adicionales.

**Estrategia de discado:** Intentos inflados de 2.2 a 7 sin criterio.  
Propuesta: discado progresivo con modelo de propensiones.

Impacto Total: +241 ventas identificadas

**Stack:** SQL (masking de datos) · Python · Power BI DAX

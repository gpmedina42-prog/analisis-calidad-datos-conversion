¿Cuánto dinero pierde una operación comercial por mala calidad de datos?
Desarrollé un análisis estratégico sobre datos reales, aplicando técnicas de masking en SQL para protección de datos sensibles antes de cualquier procesamiento o visualización.
Identifiqué 4 palancas de mejora con impacto cuantificado:
*Calidad de base: 55.6% de registros sin datos con efectividad 38% menor que registros completos (1.49% vs 2.05%). Impacto estimado: +309 ventas si toda la base fuera completa.
*Speech diferenciado: La contactabilidad es uniforme entre segmentos — el quiebre ocurre después del contacto. Implementar un speech por segmento estima +105 ventas adicionales.
*Contactabilidad: Mejora de 3 puntos porcentuales representa +87 ventas adicionales.
*Estrategia de discado: Discado predictivo sin criterio infló intentos de 2.2 a 7. Propuesta de migración a discado progresivo con modelo de propensiones.
Impacto total cuantificado: +241 ventas identificadas en 4 palancas.
Stack: SQL (masking de datos) · Python · Power BI DAX
